# 项目技术栈要求

## 核心依赖版本
- **React**: 18.3.1
- **React DOM**: 18.3.1
- **Redux Toolkit**: 1.9.7
- **redux-persist**: 6.0.0
- **TanStack Router**: 1.78.3
- **Tailwind CSS**: 3.4.1
- **包管理器**: yarn（必须使用 yarn，不要使用 npm）

## 文件扩展名要求
- **所有 React 组件文件必须使用 `.tsx` 扩展名**（不要使用 `.jsx`）
- TypeScript 配置文件使用 `.ts` 扩展名

## 配置文件
- `tsconfig.json` - TypeScript 配置，JSX 模式设置为 `"react-jsx"`（React 18 需要）
- `tailwind.config.js` - Tailwind CSS 3 配置
- `postcss.config.js` - PostCSS 配置（包含 tailwindcss 和 autoprefixer）
- `vite.config.ts` - Vite 构建工具配置

## 关键编译设置
```json
{
  "jsx": "react-jsx",
  "target": "ES2020",
  "module": "ESNext",
  "moduleResolution": "bundler"
}
```

## 样式处理
- 使用 Tailwind CSS 3 处理所有样式
- 主样式入口文件：`src/index.css`
- 包含 `@tailwind` 指令：base、components、utilities

## Redux 状态管理
- 使用 Redux Toolkit 管理应用状态
- redux-persist 自动持久化状态到 localStorage
- 存储键名：`persist:root`
- 类型安全的 hooks：`useAppDispatch` 和 `useAppSelector`

## 入口文件要求
- `src/main.tsx` - 应用主入口（使用 React 18 的 createRoot API）
- 所有组件文件必须为 `.tsx`
- 使用 Provider 包装应用以支持 Redux

## 路由管理
- 使用 TanStack Router（@tanstack/react-router）
- 包含 Router DevTools 用于调试
- 自动生成路由配置

## 依赖安装
运行以下命令安装依赖：
```bash
yarn install
```

## 开发命令
```bash
yarn dev       # 启动开发服务器（端口 3000）
yarn build     # 生产构建
yarn serve     # 预览生产构建
yarn typecheck # 类型检查
```

## 注意事项
- ❌ 不要使用 npm，必须使用 yarn
- ❌ 不要将 .jsx 文件转换为其他格式，所有文件必须是 .tsx
- ✅ 所有新创建的 React 组件都应该是 .tsx 文件
- ✅ 确保 TypeScript 严格模式启用
- ✅ Redux state 使用 redux-persist 自动持久化
- ✅ 使用 Redux Toolkit 的 slice 模式创建状态管理
